Chapter 3 — 애로사항 & 교훈

개발 중 마주친 장벽들과
돌파 방법

1
⚡ yfinance 버전 지옥 — 429 Too Many Requests
문제: requirements.txt에 yfinance==0.2.48 고정 → Yahoo Finance API 구조 변경으로 모든 가격 조회 실패. "Expecting value: line 1 column 1" 오류.
해결: yfinance 완전 배제, Yahoo Finance v8 Chart API 직접 HTTP 호출로 전환. /v8/finance/chart/{ticker}?interval=1d&range=1y → 429 완전 해소.
2
🇰🇷 한국 주식 API 엔드포인트 무단 변경
문제: Naver 모바일 API /api/stocks/top → 404, 응답 구조도 list→dict로 변경.
해결: Naver Finance PC 버전 HTML 스크래핑으로 교체. 외부 API는 언제든 변경된다 — fallback 전략 필수.
3
🖥️ PowerShell vs Bash 환경 충돌
문제: cd backend && pip install — PowerShell에서 && 미지원. taskkill 명령도 Git Bash에서 경로 충돌.
교훈: Windows 개발 환경은 명령어 호환성을 사전 확인해야 함. PowerShell / CMD / Git Bash 혼용 주의.
4
📱 Streamlit 탭 전환 — 프로그래밍 방식 불가
문제: st.tabs()는 코드에서 탭 이동 불가 → 보유탭 📈 버튼 → 차트탭 자동 이동 구현 불가.
해결: st.session_state.active_tab + st.button() 커스텀 네비게이션 구현. React 전환 후 Zustand로 완전 해소.
5
🤖 "테스트 후 보고" 원칙 — AI와의 협업 교훈
상황: AI가 코드 수정 완료 선언 → 실제 배포 없이 보고 → 사용자 화면에 반영 안됨. 이 상황 10회 이상 반복.
교훈: AI 어시스턴트를 활용할 때도 검증 기준을 명확히 사전 정의해야 함. "코드 완성 ≠ 기능 완성".

해외주식 양도소득세·배당소득세 자동 계산 기능 미구현. 국내·해외 세율 자동 적용 예정.
